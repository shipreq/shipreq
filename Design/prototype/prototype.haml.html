Notes.
  1. Two components with different goals.
    1. A language to represent *any* criteria imaginable. Goal: expressivity.
    2. A simple UI for 80% of users to easily & quickly accomplish 80%+ of typical filters. Goal: simplicity.
  2. When not editing the filter, it should take up as little space as possible. Therefore a single text representation + an edit button would probably be best.
  3. Issue filtering would be nice in the underlying expression syntax. Should not be part of the simple UI; there is a whole incompletions UI for that.

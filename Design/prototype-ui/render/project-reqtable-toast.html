Filter Help
Text
To find requirements that contain a certain word somewhere in its text, just type the word you're looking for. This is case-insensitive, so it doesn't matter whether text is upper- or lower-case.dividend
To search for a phrase or sequence of words, surround the search phrase with one of the following:
  1. quotation marks (")
  2. single quotation marks (')
  3. backticks (`)
As above, this is case-insensitive, so it doesn't matter whether text is upper- or lower-case.		"end of day"
'end of day'
`end of day`
You can also use regular expressions to search requirement text.Unlike above, this is case-sensitive unless you specify the case-insensitivity flag: (?i)./.*end[ -]*of[ -]*day.*/
/(?i).*end[ -]*of[ -]*day.*/
Issues & Tags
To find requirements that have a certain issue or tag, enter # followed by the issue or tag.#draft
You can also filter by whether a requirement has any issues or any tags by specifying has: followed by either:
  1. issues
  2. tags
As above, this is case-insensitive, so it doesn't matter whether text is upper- or lower-case.		has:issues
has:tags
Like above, you can also search for requirements that don't have any issues or tags by changing has to no.no:issues
no:tags
Req Type
To filter requirements by a certain req-type, enter the req-type's mnemonic (in upper-case).FR
Implication
To filter requirements by their implications, start with implies: or impliedBy:, then enter a list of requirements seperated by a comma (with no spaces in between).impliedBy:MF-3
implies:CO-3,FR-20
To specify a type of subject (eg. show me everything implied by BR business rules), just enter the req-type.impliedBy:BR
To specify a number of subjects of the same type, surround the numbers with braces ({...}) and separate by commas. You can also use a dash for a range.implies:FR-{1,3,5,7}
implies:FR-{10-20}
When multiple requirements are specified, the filter matches if a any match.
So impliedBy:MF-{1,2} will match everything that implied by MF-1 and everything implied by MF-2.
To match all instead of any, use multiple terms.So impliedBy:MF-1 impliedBy:MF-2 will match everything that implied by both MF-1 and MF-2.
impliedBy:MF-1 impliedBy:MF-2
Finally, all of the above abilities can be combined; separate by comma as usual.impliedBy:MF,FR-{1-10,20,30},CO-7
Negation & Combination
To negate a clause, enter a minus (-) before it.
This works for anything, including text searches.		-dividend
-#draft
-impliedBy:MF
-(#v1.0 #released)
To specify multiple mandatory criteria, separate them by a space. Every criterion will need to match for a requirement to be allowed by the filter.Optionally, you can wrap them in parenthesis ((...)) and treat it as a single clause which can then be negated.#v1.0 #released
(#v1.0 #released)
While the above requires that all criteria need match, you can also specify that any need match. In order words, at least one criterion will need to match.
To do so, wrap criteria in braces ({...}).
For example, {#v1.0 #v1.1} will match:
  • requirements tagged with v1.0
  • requirements tagged with v1.1
  • requirements tagged with v1.0 and v1.1
{MF FR}
{has:issues #bug}
Advanced Examples
The example to the right selects all released functional requirements (FR) and use cases (UC), that either:
  • have open issues but no #bug tag.
  • don't have an implying major feature (MF).
#released {FR UC}
{(has:issues -#bug) -impliedBy:MF}
Rich Text Editor Help
References
Requirements can be referenced by putting the ID inside square brackets.[FR-3]
Use case steps can be referenced by putting the use case ID and step number, inside square brackets.[UC-2.0.3.a]
Codes can be referenced by putting the code inside square brackets.[backend.backup.times]
Issues & Tags
Issues can be declared in text by typing a hash (#) followed by the issue.#issue
Tags can be declared in text by typing a hash (#) followed by the tag.#tag
Multiline
A list of bullet points can be created by starting new lines with an asterix (*) followed by a space.* item 1
* item 2

Mathematical expressions can be entered in TeX format, by surrounding in <math>...</math>.

For more detail, see KaTeX or Symbols and Functions in KaTeX.		<math>{1 \over n} + x^2</math>

Links & References

Referencing a Step

In order to refer to a step, type the step number with square brackets around it into your text.

Example: This is a reference to [1.0.1].

If the step number later changes, the reference will be updated automatically.

Referencing a Use Case

In order to refer to a use case, type [UC123] or [UC-123] where 123 is the number of the use case you want to refer to. The system will verify that the use case number is valid, and insert the use case's title.

Example, you type:

Refer to [UC4].
The system will validate UC-4 and replace it with this:
Refer to [UC-4: Manager requests Month-End Report].

Step Flow

You can indicate execution paths by typing an arrow proceded by one or more steps, at the end of a step's text.

--> indicates that the current step flows to other steps.
<-- indicates that other steps flow to the current one.

Example, you type:

This jumps forward to another step. --> 1.4
The system will validate and replace it with:
This jumps forward to another step. ➡ [1.4]

As with normal step references, if the step numbers change your flow will be updated automatically.

Invalid References

If you enter a reference for something that doesn't exist, a question mark will be added to it.

Example: The reference [1.8?] is invalid.

If the subject of a valid reference is later deleted, then those references will be automatically changed to [DELETED].

Keyboard Shortcuts

When typing, you can press the following key combinations to get things done faster.
Alt + EnterCreate next step.
Alt + Up/DownMove cursor to previous/next field.
Alt + Left/RightIncrease/decrease step indent.

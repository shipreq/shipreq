Hello and welcome to the ShipReq Blog.

## What is "ShipReq"?
Well if you're reading this you're likely a fellow techie, and probably expecting a technical article. <ShipReq/>
is a service that aims to drastically improve life for teams (or even individuals)
developing software. This post isn't a sales pitch, so if you're interested,
feel free to head over to the <About/> page for more. Otherwise, I'll introduce it from a technical perspective.
It's a SAAS built from the ground-up with FP (functional programming); both in the backend, which is written in Scala;
and in the frontend, which is written in ...can you guess...? It's written in Scala too using <ScalaJS/>, which
transpiles and optimises Scala into plain old JS. The generated JS is incomprehensible to humans but luckily
source maps can also be generated.
Deciding to use Scala for both the frontend as well as the backend was a decision that felt a bit risky at first,
but has paid dividends, daily, for years, and I couldn't be happier.
Being able to effortlessly share, and move code around between the frontend and backend is profoundly beneficial.

There are a bunch of satellite components and services in the ShipReq ecosystem if you peek behind the covers,
most of which are third-party OSS apps.
Notably, <A href="https://prometheus.io">Prometheus</A>
and <A href="https://grafana.com/oss/grafana">Grafana</A>
deserve an applause, and I highly recommend you give them a look if you don't know what they are.

All infrastructure is managed by <A href="https://www.terraform.io">Terraform</A>.
Finally with my own venture I'm in a position where if I decide I wanna spin up a completely
new environment, complete with metrics and security and scalability etc etc, I basically just
hit a button and enjoy a 20 min coffee break. Sooo many times in my career when employed for others have I
(and the entire team) been frustrated that we couldn't spin up a new UAT cluster, or do performance testing
without locking everyone else out.

This site is written
with <A href="https://www.gatsbyjs.org">Gatsby</A>, <A href="https://www.typescriptlang.org">Typescript</A>, <A href="https://mdxjs.com">MDX</A> <span style="opacity:0.5">(= MD + JSX)</span>
which, by the way, has been a surprisingly awesome experience.
Typescript is still a pretty weak type-system than I'm used to but it covers the 80 (%) very well.
I mean, blogs aren't complex so really it covers the 97% very nicely.


## What is this blog for?

There are two types of content you can expect to see here.

1. **Service related:** articles discussing <ShipReq/>, the service.

  Anything from announcements of, or deep-dives into new features, to notifications of uptime disruption.

2. **Technical posts:** educational material, mostly for programmers.

  There's a reason I listed a bunch of technologies above.
  When I come across something I think is an interesting discovery, invention, workaround, whatever, in those domains,
  if I judge it to be the right combination of valuable and not-well-known then this is where you can find me posting about it.

Phase #1: Project Start
=======================

`phase-1/UCE-HighLevelReq.ods`
  * Envisioned models and functions for UCE 1.0.
  * Incomplete and differs from actual 1.0: -shares, +quotas, +subscriptions.

`phase-1/UCE-UCs.ods`
  * Use cases for UCE 1.0.
  * Massively incomplete. Abandoned due to frustrations of manual UC authoring.

`phase-1/UCE-mockups.ods`
  * UI mockups for UCE 1.0.
  * Massively incomplete. Abandoned.

`phase-1/requirements-all.md`
  * Brainstorm of all envisioned features.
  * Benefits of product.
  * Quality attributes.

`phase-1/requirements-shortlist.md`
  * Feature subset under consideration.

`phase-1/use_case_editor.md`
  * Imagined UCE prototype.

Phase #2: Post UCE 1.0
======================

`major_features.ods`
  * List of major features envisioned.
  * Version and analysis/impl status for each feature.
  * Analysis of feature intersection.

`ideas.md`
  * Quick-and-easy dumping ground for project ideas. Predates `major_features.ods` which is a subset.

`analysis-mf*`
  * Analysis of a Major Feature.
  * Extraction of initial requirements from `interview-nnn` files.

`interviews.ods`
  * Summary of contents of `interview-nnn` files.

`interview-nnn.odt`
  * Requirements gathering interviews. (prelim & elicitation)

`project-attributes.ods`
  * Top 5 project attributes.
  * Definition and analysis of each attribute.
  * Prioritisation of attributes.

`reqs.ods`
  * Requirements list.

`UX-mf*.uml`
  * A sketch of the UX when the user performs a function pertaining to the Major Feature indicated in the filename.
    A state diagram is used and should be interpreted a little differently:
    The top half of a state is what the user sees on the screen.
    The bottom half of a state is what the user wants to do once on the screen.

`UX-sitemap-current.gv`
  * The sitemap as it is currently implemented.

`UX-sitemap-future.md`
  * Ideas about what the sitemap will be like when more Major Features are added.
